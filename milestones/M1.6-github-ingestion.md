### 1.6.0 — Migrate from GitHub OAuth App → GitHub App

> **Why:** GitHub Apps show a native repo-selection screen during installation ("All repositories" or "Only select repositories"). OAuth Apps give no such control — it's all-or-nothing. This is the same mechanism Vercel, Netlify, and Railway use. The migration touches `auth.py` and `github_service.py` only — everything else is unchanged.

**Step 1 — Register a GitHub App**

> This replaces the existing GitHub OAuth App entirely. No separate OAuth App registration is needed — the GitHub App's own Client ID/Secret handles user identity as well as repo access.

- [ ] Go to GitHub → Settings → Developer Settings → **GitHub Apps** → New GitHub App
- [ ] Set:
  - **App name:** `CareerForge`
  - **Homepage URL:** your frontend URL
  - **Callback URL:** `{BACKEND_URL}/api/auth/github/callback` (same URL as before — callback logic barely changes)
  - **Webhook:** disable (uncheck "Active") — not needed
  - **Repository permissions:** `Contents: Read-only`, `Metadata: Read-only`
  - **User permissions:** `Email addresses: Read-only`
  - **Where can this app be installed:** Any account
  - ✅ **"Request user authorization (OAuth) during installation"** — **check this box**
    > This is the key setting. GitHub will automatically chain the OAuth consent screen after the user selects repos, so identity (email, name, avatar) and repo access are captured in a single flow. No `/installed` redirect endpoint needed. No separate OAuth App needed.
  - **Setup URL:** leave blank (not needed when the above checkbox is enabled)
- [ ] After creation, note down: **App ID**, **Client ID**, **Client Secret**
- [ ] Generate and download **Private Key** (`.pem` file) — this signs JWT tokens to get installation access tokens
- [ ] Store private key in **AWS Secrets Manager**:
  ```bash
  aws secretsmanager create-secret \
    --name careerforge/github-app-private-key \
    --secret-string "$(cat your-app.private-key.pem)"
  ```
- [ ] Add to `.env`:
  ```

  GITHUB_APP_SLUG=careerforge
  GITHUB_APP_CLIENT_ID=Iv1.xxxxxxxx

  import jwt as pyjwt   # pip install PyJWT
  import time
  import boto3

  async def get_installation_token(self, installation_id: int) -> str:
      """Generate an installation access token scoped to repos the user selected."""
      # Load private key from Secrets Manager
      sm = boto3.client("secretsmanager", region_name=settings.AWS_REGION)
      pem = sm.get_secret_value(SecretId=settings.GITHUB_APP_PRIVATE_KEY_SECRET)["SecretString"]

      # Sign a 10-min JWT as the GitHub App
      now = int(time.time())
      payload = {"iat": now - 60, "exp": now + 540, "iss": settings.GITHUB_APP_ID}
      app_jwt = pyjwt.encode(payload, pem, algorithm="RS256")

      # Exchange for an installation access token
      async with httpx.AsyncClient() as client:
          r = await client.post(
              f"https://api.github.com/app/installations/{installation_id}/access_tokens",
              headers={
                  "Authorization": f"Bearer {app_jwt}",
                  "Accept": "application/vnd.github+json",
              },
          )
          r.raise_for_status()
          return r.json()["token"]   # short-lived token, valid 1hr
  ```

### 1.6.2 — Add `POST /api/github/ingest` Endpoint

**File:** `project/backend/app/api/routes/projects.py`

> M1.5 scaffolded the "Refresh Repos" button to call `POST /api/github/ingest`. This endpoint does not exist yet — there is only `POST /api/projects/import-github` which uses SQLAlchemy. Add the new DynamoDB-native endpoint.

- [x] Add a new file or route: `project/backend/app/api/routes/github.py` with router prefix `/api/github`
- [x] `POST /api/github/ingest` — full ingestion pipeline:
  ```python
  @router.post("/ingest")
  async def ingest_github_repos(
      include_forks: bool = True,   # include forks — users may have contributed significantly
      current_user = Depends(get_current_user_dynamo),
  ):
      user_id = current_user["userId"]
      github_token = current_user.get("githubToken")
      installation_id = current_user.get("githubInstallationId")
      if not github_token:
          raise HTTPException(400, "No GitHub token. Please re-connect GitHub.")
      
      # Mark ingestion as in_progress
      await dynamo_service.update_item(
          table=f"{settings.DYNAMO_TABLE_PREFIX}Users",
          key={"userId": user_id},
          updates={"ingestionStatus": "in_progress"},
      )
      
      try:
          # Fetch user-selected repos via installation token
          repos = await github_service.fetch_user_repos_fast(
              encrypted_token=github_token,
              installation_id=installation_id,
              include_forks=include_forks,
          )
          
          processed, failed = 0, 0
          for repo_meta in repos:
              try:
                  import asyncio
                  await asyncio.sleep(0.1)  # respect rate limits
                  # Fetch README, dep files, root dirs
                  repo_detail = await github_service.fetch_repo_details(
                      full_name=repo_meta["full_name"],
                      encrypted_token=github_token,
                  )
                  # Bedrock structured summary → .md content
                  project_data = await github_service.create_project_from_repo(repo_detail)
                  # Upload .md to S3, persist to DynamoDB
                  project_id = str(uuid.uuid4())
                  await github_service.ingest_and_embed_repo(
                      repo_data=repo_detail,
                      project_data=project_data,
                      user_id=user_id,
                      project_id=project_id,
                  )
                  processed += 1
              except Exception as e:
                  logger.warning("Failed to ingest repo", repo=repo_meta["name"], error=str(e))
                  failed += 1
          
          # Update ingestion status
          summary = {"total": len(repos), "processed": processed, "failed": failed, "lastRunAt": datetime.utcnow().isoformat()}
          await dynamo_service.update_item(
              table=f"{settings.DYNAMO_TABLE_PREFIX}Users",
              key={"userId": user_id},
              updates={"ingestionStatus": "done", "ingestionSummary": summary},
          )
          return {"status": "done", **summary}
      
      except Exception as e:
          await dynamo_service.update_item(
              table=f"{settings.DYNAMO_TABLE_PREFIX}Users",
              key={"userId": user_id},
              updates={"ingestionStatus": "failed"},
          )
          raise HTTPException(500, f"Ingestion failed: {e}")
  ```

## Notes

- **No separate OAuth App needed:** The GitHub App with "Request user authorization during installation" handles both repo access and user identity. The App's own Client ID/Secret replace the old `GITHUB_CLIENT_ID`/`GITHUB_CLIENT_SECRET`. Delete the old OAuth App from GitHub Developer Settings after this ships.
- **GitHub App vs OAuth App:** GitHub App installation token is scoped to only the repos the user selected. The installation expires and auto-rotates. OAuth `repo` scope was overkill and a privacy liability.
- **Private key security:** Never commit the `.pem` file. It lives only in AWS Secrets Manager. The `get_installation_token()` call fetches it on every ingest — acceptable for hackathon; cache it in prod.
- **Installation token TTL:** 1 hour. For large ingestions (50+ repos × fetch_repo_details), a single token is sufficient within that window.
- **Do not port the entire `projects.py` to DynamoDB in this milestone.** Scope is minimal: GET by userId + the ingest endpoint. Everything else stays SQLAlchemy until M6 (cleanup).
- **Ingestion signal sources per repo:** README (3000 chars) + dependency files + GitHub language breakdown + top-level folder names (fallback only). No source code files read — not needed and would explode token cost.
- **Dropped fields:** `forks` (no signal), `isPrivate` (installation scopes this already), `commit_count` (volume ≠ quality).
- **No vector storage at ingestion time:** ChromaDB and Titan embeddings are removed. M2 passes all `.md` summaries directly to Claude, which does its own intelligent project ranking via the Step 0 prompt analysis (see `new-resume.prompt.md`). On-demand embedding is deferred to post-hackathon if ever needed.
- **Rate limits:** GitHub API allows 5,000 authenticated requests/hour. `fetch_repo_details()` costs 3–5 requests per repo. With 50 repos at 5 req/repo = 250 requests — well within limits. 0.1s sleep between repos is already in the ingest loop.
- **Cost:** Bedrock Haiku at ~$0.25/M input tokens. A 2000-char README ≈ 500 tokens. 50 repos = 25,000 tokens ≈ $0.006. Not a concern.
- **Don't delete `gemini_client.py`** — it's imported by old SQLAlchemy routes that haven't been ported yet. Clean-up is M6.
