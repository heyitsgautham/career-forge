## Changes Implemented

### Area A — GitHub Import Modal

#### A1: Visibility Filter (All / Public / Private)

**File:** [projects-list.tsx](../../project/frontend/src/components/dashboard/projects-list.tsx)

- Added `visibilityFilter` state with 3 options: `'all' | 'public' | 'private'`
- Extended `filteredRepos` memo to filter by `is_private` flag
- Added 3-button toggle UI with count badges: `All (N) | Public (N) | Private (N)`
- Counts are computed from the full `userRepos` list before filtering

#### A2: Projects Loading Banner

- Added `projectsLoading` prop to `GithubImportModal`
- Shows "Loading your imported projects…" banner with spinner when `projectsLoading` is true
- Prevents false negatives in "Imported" badge detection

#### A3: Per-Repo Progress Status List

- Replaced flat `importingRepos: string[]` with structured `importProgress` array:
  ```ts
  Array<{ name: string; fullName: string; status: 'queued' | 'processing' | 'done' | 'failed' }>
  ```
- Each repo shows individual status with appropriate icons and colors
- Simulated sequential progress animation (Phase 1 — visual only)
- Status updates to `done` or `failed` based on API response

#### A4: Non-Blocking Import Banner

- Converted blocking `ImportLoadingOverlay` to a compact `ImportProgressBanner`
- Banner shows at the top of Projects tab without replacing content
- Visual indicators: dot progress indicator for each repo status
- Users can continue viewing their projects while imports run

#### A5: Post-Import Polling

- Added 30-second polling loop after successful import:
  ```ts
  const pollId = setInterval(() => {
    queryClient.invalidateQueries({ queryKey: ['projects'] });
  }, 5000);
  setTimeout(() => clearInterval(pollId), 30_000);
  ```
- Ensures newly imported projects appear without manual refresh

### Area B — Dashboard Sidebar Scroll Isolation

**File:** [dashboard/page.tsx](../../project/frontend/src/app/dashboard/page.tsx)

#### B1: Sticky Sidebar Layout Fix

**Root cause:** The outer container used `flex min-h-screen`, allowing the document to grow beyond the viewport. `<aside>` had `relative` positioning, causing it to stretch to full document height and scroll with the main content.

**Changes made:**

1. **Outer container** — `flex min-h-screen` → `flex h-screen overflow-hidden`  
   Locks the layout to exactly the viewport height; eliminates document-level scroll

2. **Sidebar** (`<aside>`) — `relative flex flex-col` → `sticky top-0 h-screen flex flex-col overflow-hidden`  
   Sidebar occupies exactly one viewport height and stays anchored regardless of content scroll

3. **Main content** (`<main>`) — added `overflow-y-auto` directly on `<main>`; removed it from inner content `<div>`  
   Makes `<main>` the scroll container, which is required for any `sticky`-positioned elements inside it (e.g. sticky headers) to work correctly

```tsx
// Before
<div className="flex min-h-screen">
<aside className="... relative flex flex-col">
<main className="flex-1 flex flex-col min-w-0">
<div className="flex-1 overflow-y-auto p-6 animate-fade-in-up">

// After
<div className="flex h-screen overflow-hidden">
<aside className="... sticky top-0 h-screen flex flex-col overflow-hidden">
<main className="flex-1 min-w-0 flex flex-col overflow-y-auto">
<div className="flex-1 p-6 animate-fade-in-up">
```
