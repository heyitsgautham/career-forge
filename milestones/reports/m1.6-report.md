## Architecture Decisions

1. **GitHub App over OAuth App**: Installation tokens scoped to user-selected repos only (no noisy 200-repo problem). Single flow: install + OAuth combined via "Request user authorization during installation".

2. **Structured Bedrock summaries**: `_generate_structured_summary()` produces a JSON dict with `oneLiner`, `problemType`, `domain`, `languages`, `frameworks`, `infrastructure`, `keyTechniques`, `capabilities`, `highlights`. This gives M2's resume generator rich, structured input.

3. **No vector storage**: ChromaDB and Titan embeddings removed. M2 passes `.md` summaries directly to Claude for project ranking. On-demand embedding deferred to post-hackathon.

4. **S3 + DynamoDB dual storage**: `.md` files in S3 for human-readable summaries and M2 consumption. Structured items in DynamoDB `Projects` table for API queries and frontend display.

5. **Background ingestion**: New users get immediate `pending` status and background `asyncio.create_task()`. Frontend auto-polls on mount if status is active.

6. **Graceful fallbacks**: No README → `_fetch_root_dirs()` provides folder names as context. Bedrock failure → minimal fallback summary using repo metadata. No installation token → falls back to OAuth token path.
