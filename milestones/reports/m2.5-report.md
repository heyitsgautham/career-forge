## Summary

Implemented the full M2.5 in-browser LaTeX resume editor. Users can now open any generated resume and get a full IDE-like editing experience: Monaco editor on the left, live PDF preview on the right, and a Claude-powered AI chat drawer at the bottom. Three new backend endpoints were added, the Bedrock client gained SSE streaming, and four new frontend components were created to replace the old basic editor page.

## Known Limitations & Future Work

- **Error line markers**: The `addErrorMarkers()` API is wired but the backend currently returns an error at line 0 only. Parsing LaTeX stderr for specific line numbers (e.g. `l.42 Undefined control sequence`) and forwarding them to the frontend is a small follow-up task.
- **Chat history persistence**: In-memory only — lost on page refresh. DB persistence deferred per the spec.
- **Auto-compile ON behavior**: When enabled, the 3s debounce fires compile on every keystroke. For large `.tex` files this can feel slow; a future improvement would be a smarter dirty-check before triggering.
- **`PUT /latex` vs `PATCH /latex`**: Both endpoints now exist. The old `PATCH` was kept for backwards compatibility with any existing clients. New code uses `PUT`.
